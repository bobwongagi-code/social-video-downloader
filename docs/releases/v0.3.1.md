# v0.3.1 Release Notes

Suggested GitHub release title:

```text
v0.3.1 - Public repo baseline, governance docs, and CI
```

Suggested GitHub release body:

```markdown
## Summary

`v0.3.1` establishes the first clean public-repository baseline for Social Video Downloader.

This release turns the project from a private working tool into a more shareable open-source repository with:

- A stable downloader flow for social pages, direct media URLs, and HLS inputs
- Practical defaults for balanced quality, audio-preserving output, and PowerPoint/QuickTime compatibility
- Public-facing repository docs, contribution guidance, security guidance, and MIT licensing
- A lightweight repository validation script plus GitHub Actions CI

## Highlights

### More stable real-world download behavior

- Social-page URLs and direct media URLs follow separate download paths
- HLS downloads support a fast path plus segmented fallback recovery
- Audio-only restricted sources are treated as failures instead of false success
- Output is normalized for `H.264 + AAC` playback only when needed

### Better repeated-work workflows

- Reuses cached successful downloads when the cached file is still valid
- Logs lightweight KPI data for recent real runs
- Supports bounded concurrency for multi-URL batches

### Public repository readiness

- Added `README.md`, `CHANGELOG.md`, `CONTRIBUTING.md`, and `SECURITY.md`
- Added MIT `LICENSE`
- Added a repo-local validation script at `scripts/validate_repo.py`
- Added GitHub Actions CI for basic repository checks

## Validation

The release baseline was checked with:

```bash
python3 -m py_compile scripts/download_social_video.py scripts/validate_repo.py
python3 scripts/validate_repo.py
git diff --check
```

## Notes

- This project is not affiliated with TikTok, Instagram, Facebook, X/Twitter, YouTube, or Xiaohongshu
- Platform support still depends on the current `yt-dlp` extractor and the target site's restrictions
- Users are responsible for complying with platform terms and local laws when downloading content
```
